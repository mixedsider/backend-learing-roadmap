# Commit Message Examples

좋은 커밋 메시지는 "무엇을 왜 바꿨는지"를 짧고 분명하게 말합니다.

```text
feat: add signup endpoint
fix: return 404 when todo does not exist
docs: update database setup guide
test: add auth service unit tests
refactor: separate user repository
chore: update gitignore
```

## 추천 형식

```text
type: short summary
```

자주 쓰는 type:

- `feat`: 기능 추가
- `fix`: 버그 수정
- `docs`: 문서 수정
- `test`: 테스트 추가/수정
- `refactor`: 동작 변경 없는 구조 개선
- `chore`: 설정, 빌드, 기타 작업
